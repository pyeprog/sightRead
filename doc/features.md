## 二、当前功能包（核心）

全部为本地静态能力（L0/L1），毫秒级响应，零 LLM 依赖。

### 2.1 函数段落化（#1–#4）

一份数据（段落 = 命名的、带角色的范围）点亮多处 UI：

- **#1 手动划段**：选中若干行 → 命名为段落（"corner case 守卫""主算法"…）。段落名以行尾标签渲染（decoration `after`），段落体淡色背景。
- **#2 自动切段**：无 LLM，基于空行、注释、控制流形状（early return、循环、try/catch）给候选，人来审阅修正。**已知风险**：切段质量跟随代码质量，烂代码上启发式会失灵——所以自动只是"候选"，手动（#1）才是兜底。
- **#3 段落注入 `DocumentSymbolProvider`**：Outline、面包屑、Sticky Scroll 三处原生 UI 同时获得段落感知。
- **#4 角色配色**：守卫 / 主逻辑 / 错误处理 / 日志等角色各一色系，函数变成语义热图。

### 2.2 聚光灯（#5）与变量读写染色（#15）

- **#5 段落聚光灯**：光标所在段落正常显示，其余 `opacity 0.25` 置灰，光标移动自动切换焦点。依赖段落数据（#1/#2）。
- **#15 变量读写染色**：光标落在变量上（或快捷键触发），函数内该变量的**写入点一色、读取点一色**，一眼看出值的生命周期形状（"赋值一次到处读" vs "循环里反复累加"是完全不同的形状）。
- 两者可视为同一个"聚光灯"机制的两种焦点模式：**段落焦点**与**变量焦点**，实现上共享置灰/点亮基础设施。

### 2.3 多色荧光笔（#8）

任意选区涂色（黄=重点、红=存疑、绿=已验证），gutter 图标 + overview ruler 彩条同步显示，不碰源码。纯装饰实现（`createTextEditorDecorationType`），无依赖。

### 2.4 骨架折叠（#6，待原型实证）

纯语法驱动，不依赖段落标注：折叠内联函数定义、if/else 体、循环体、catch 体，其余不折。折叠"永远保留首行"的限制在此恰是特性——`if (cond)` 行保留，分支结构一目了然。与原生"按层级折叠"（Ctrl+K Ctrl+2）的差异：**按语法构造类型选择**，而非按缩进层级。原型一两百行，效果好坏做出来在真实代码上一试便知。

## 三、次要池（以后再说）

- **#10 变量别名透镜**：给难懂变量挂"我的理解名"，inlay hint 内联显示。
- **#11 疑问清单**：存疑标注汇总为侧边栏列表，本质是书签变体。
- **#14 peek 式集合浏览**：标注集合用 `peekLocations` 浮窗浏览。

## 四、暂缓（#17 + #21）

调用点摘要内联的价值成立（读调用方时不必跳进跳出），但被速度问题卡死：LLM 必须读完足够上下文才能理解并 summarize 一个函数，无论 hover 时现算还是预先批量生成，都太重太慢。**暂缓，不进当前版本。** 若将来重启：接口形态定为纯 API（VS Code 命令），不引 Claude Code、不做 MCP；摘要按需生成。

## 五、由取舍推导出的设计原则

1. **标注是短命的，服务于当前这轮阅读。** F 组（体检/重读 diff/导出）整体去掉 + #9 的否决理由，共同指向：不追求标注长存、不做跨读者分享、不做过期检测。数据观：一轮阅读（一个 session 到几天）内有效，过期即弃。存储设计可以因此保持极简。
2. **不做任何形式的导航图。** 函数级小径（人为踩出的导航图）也砍掉，与"宏观导航不是目标"的边界完全一致。
3. **当前版本 100% 本地静态。** LLM/agent 相关（#17/#21/#22）全部暂缓或去掉，插件不含任何网络/模型依赖——与 discussion.md 6.2"尽量不引入 LLM"的倾向一致，且比原设想更彻底。

## 六、第三轮修订（设计讨论后的增量结论）

设计讨论（详见 [design.md](./design.md)）对上表的修正：

- **#1 手动划段：废弃**。悖论：手动划完段 = 已经读完读懂 = 不再需要辅助。自动切段（#2）成为段落数据的**唯一来源**，信号 = 空行 + 顶层控制流块 + 注释绑定下一段。
- **#4 角色配色：废弃**。角色是语义的，自动给不了；将来可考虑纯语法版（guard/循环/错误处理）用左边框色条，v1 不做。
- **段落名 = 段首行代码文本**（截断），不编造语义；#3 Outline 注入降为**实验性开关（默认关）**，provider 合并行为待实证。
- **#6 骨架折叠简化为一对命令**：fold = 函数内所有可折叠块从顶层递归折起（逐层点开渐进展开）；unfold = 还原。不注册 provider。
- **#8 荧光笔补充**：说明文字放首行行尾；删除双保险（编辑相交自动删 + 批量手动删）；新标记吞掉相交旧标记。
- **#15 变量染色定形**：与荧光笔构成对偶（自动/手动、瞬时/持久、推断/选定、描边/填充）。
- **#5 聚光灯定形为三档**：函数 → 段落 → 段落+变量，逐档退化，垃圾代码自动降为一档。
